# .NET demo

Calls the WeaveFFI C ABI from C# via P/Invoke and prints the SHA-256 of
the processed PNG. Should match every other language's hash exactly.

## Prerequisites

- .NET 8 SDK (`brew install --cask dotnet-sdk` or
  https://dotnet.microsoft.com/download)
- The shared library built from the workspace root:
  ```sh
  bash scripts/build-rust.sh
  ```

## Run

```sh
bash demos/csharp/run.sh
```

This sets `DYLD_LIBRARY_PATH` so `[DllImport("weaveffi")]` finds
`libweaveffi.dylib` from the cargo target dir.

## Note on the generated SDK

`sdk/dotnet/WeaveImage.cs` is generated by WeaveFFI but currently has a
P/Invoke signature mismatch in `Image.Process` (passes `Operation[]`
where the native expects `IntPtr ops, UIntPtr ops_len`). This demo uses
direct P/Invoke instead. Once the upstream fix lands, swap to:

```csharp
using WeaveImage;
using var resize = Image.Resize(512, 512);
using var blur   = Image.Blur(2.0);
using var gray   = Image.Grayscale();
var output = Image.Process(input, new[] { resize, blur, gray }, ImageFormat.Png);
```
